# Deploying prompts.chat to Nhost

This guide explains how to deploy prompts.chat to [Nhost](https://nhost.io), a backend platform that provides PostgreSQL, GraphQL (Hasura), Authentication, Storage, and Serverless Functions.

## Prerequisites

- A [Nhost account](https://app.nhost.io/signup)
- [Nhost CLI](https://docs.nhost.io/platform/cli) installed (optional, for local development)
- Git repository connected to Nhost

## Quick Start

### 1. Create a New Nhost Project

1. Go to [Nhost Console](https://app.nhost.io)
2. Click "Create Project"
3. Choose a region closest to your users
4. Wait for the project to be provisioned

### 2. Connect Your Repository

1. In the Nhost Console, go to your project settings
2. Navigate to "Git" or "Deployments"
3. Connect your GitHub repository: `solution8-com/S8-Utility-Promptschat`
4. Select the branch you want to deploy (e.g., `main`)
5. Nhost will automatically detect `nhost.toml` and deploy

### 3. Configure Environment Variables

In the Nhost Console, navigate to Settings → Environment Variables and add:

**Required:**
```
DATABASE_URL=<automatically-provided-by-nhost>
HASURA_GRAPHQL_ADMIN_SECRET=<auto-generated>
AUTH_CLIENT_URL=https://your-frontend-url.com
NEXTAUTH_URL=https://your-frontend-url.com
NEXTAUTH_SECRET=<generate-a-secure-random-string>
```

**Optional OAuth Providers:**
```
AUTH_GITHUB_ID=<your-github-oauth-client-id>
AUTH_GITHUB_SECRET=<your-github-oauth-client-secret>
AUTH_GOOGLE_ID=<your-google-oauth-client-id>
AUTH_GOOGLE_SECRET=<your-google-oauth-client-secret>
AUTH_AZURE_AD_CLIENT_ID=<your-azure-ad-client-id>
AUTH_AZURE_AD_CLIENT_SECRET=<your-azure-ad-client-secret>
AUTH_AZURE_AD_TENANT_ID=<your-azure-tenant-id>
```

**Optional Features:**
```
OPENAI_API_KEY=<your-openai-api-key>
S3_BUCKET=<your-s3-bucket>
S3_REGION=<your-s3-region>
S3_ACCESS_KEY_ID=<your-s3-access-key>
S3_SECRET_ACCESS_KEY=<your-s3-secret-key>
```

### 4. Deploy

Push your code to the connected branch:

```bash
git push origin main
```

Nhost will automatically:
1. ✅ Detect `nhost.toml`
2. ✅ Run database migrations from `nhost/migrations/`
3. ✅ Apply Hasura metadata from `nhost/metadata/`
4. ✅ Deploy serverless functions from `functions/`
5. ✅ Set up authentication and storage

### 5. Access Your Application

After deployment completes:

- **GraphQL API:** `https://your-subdomain.nhost.run/v1/graphql`
- **Functions:** `https://your-subdomain.nhost.run/v1/functions`
- **Storage:** `https://your-subdomain.nhost.run/v1/storage`
- **Auth:** `https://your-subdomain.nhost.run/v1/auth`

## Deployment Components

### 1. Database Migrations (`nhost/migrations/`)

Migrations are automatically applied in order during deployment. Each migration folder contains:
- `up.sql` - Schema changes to apply

The migrations are sourced from Prisma migrations and converted to Hasura format.

### 2. Hasura Metadata (`nhost/metadata/`)

Metadata defines:
- GraphQL schema and relationships
- Permissions and roles
- Actions and remote schemas
- Event triggers and cron jobs

### 3. Serverless Functions (`functions/`)

TypeScript/JavaScript functions that become HTTP endpoints:
- `functions/health.ts` → `/v1/functions/health`
- `functions/hello.ts` → `/v1/functions/hello`

## Local Development

### Install Nhost CLI

```bash
npm install -g nhost
```

### Start Local Backend

```bash
nhost dev
```

This starts:
- PostgreSQL on `localhost:5432`
- Hasura on `http://localhost:1337`
- Nhost Auth
- Nhost Storage
- Serverless Functions

### Connect Next.js to Local Backend

Update your `.env.local`:

```bash
DATABASE_URL="postgresql://postgres:postgres@localhost:5432/local"
NEXTAUTH_URL="http://localhost:3000"
```

Start Next.js:

```bash
npm run dev
```

## Database Management

### Run Migrations

Migrations run automatically on deployment. To run manually:

```bash
# Using Nhost CLI locally
nhost db migrate apply

# Or using Hasura CLI
hasura migrate apply --endpoint https://your-subdomain.nhost.run
```

### Access Database

Get connection string from Nhost Console:
1. Go to Settings → Database
2. Copy the connection string
3. Use with your preferred PostgreSQL client

Or use Hasura Console:
```
https://your-subdomain.nhost.run/console
```

## Troubleshooting

### Deployment Failed

Check deployment logs in Nhost Console under "Deployments" tab.

Common issues:
- **Migrations failed:** Check SQL syntax in migration files
- **Metadata failed:** Verify YAML syntax in metadata files
- **Functions failed:** Check TypeScript compilation errors

### Migration Conflicts

If migrations are out of sync:
1. Export current database schema
2. Resolve conflicts manually
3. Create a new migration with fixes

### GraphQL Schema Issues

1. Go to Hasura Console
2. Check "Data" tab for table relationships
3. Verify permissions under "Permissions" tab
4. Re-export metadata if needed

## Continuous Deployment

### Automatic Deployments

Nhost automatically deploys when you push to the connected branch:

```bash
git add .
git commit -m "Update feature"
git push origin main
```

### Preview Deployments

Create preview deployments for pull requests:
1. Enable in Nhost Console → Settings → Git
2. Each PR gets a temporary environment
3. Test changes before merging

## Security Best Practices

1. **Never commit secrets** - Use Nhost environment variables
2. **Use strong admin secrets** - Generate with `openssl rand -base64 32`
3. **Configure CORS properly** - Set allowed domains in `nhost.toml`
4. **Set up row-level security** - Define permissions in metadata
5. **Enable Hasura admin secret** - Always set in production

## Scaling

Nhost automatically scales:
- Database connections (up to 50 default)
- Serverless functions (auto-scaling)
- GraphQL queries (load balanced)

For custom scaling:
1. Go to Settings → Resources
2. Adjust compute and storage
3. Save changes
